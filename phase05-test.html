Phase 05 · Test

Collect real feedback and iterate for improvement.

In this final phase, your team tests the prototype with real target users, analyses feedback patterns, improves the solution direction and submits Gate 3 before portfolio completion.

What is Test?

Test is where your team validates the prototype with real target users. The goal is not to defend your design, but to observe how users react, where they struggle, what works, and what should be improved before final portfolio submission.

What students need to do

  • Select at least 3 real target users who match your Persona.
  • Prepare simple test tasks and set up the test environment.
  • Run user tests using T14 User Feedback Form — one record per tester.
  • Observe silently and avoid explaining how the prototype works.
  • Analyse feedback patterns and identify improvement opportunities.
  • Create T15 Improvement Plan and T16 Final Reflection.
  • Submit Gate 3 for final supervisor approval before portfolio completion.

Iterate for improvement

1. ObserveWatch users try the task.
2. PatternFind repeated issues.
3. ImprovePlan fixes and priorities.
4. ReflectConnect learning to the DT journey.

Common mistakes to avoid

  • Do not explain the prototype before users try it. Their confusion is useful evidence.
  • Do not defend your design when users struggle. Stay silent and observe.
  • Do not test only with friends who will say nice things.
  • Do not rely on memory. Write observations immediately.
  • Do not treat negative feedback as failure. It is the strongest source of improvement.

A14User Test Rules

Explain the purpose, not the solution.

Ask the tester to think aloud while using the prototype.

Observe behaviour, hesitation and confusion moments.

Use tester codes such as U01, U02 and U03 instead of real names.

Sample Test Outcome

Canteen crowd-checking prototype

Task: "Find which stall has the shortest queue before lunch break."
Observation: Tester understood the colour labels but hesitated at the refresh button.
Quote: "I like the crowd level, but I need to know when it was last updated."
Suggestion: Add a "last updated" timestamp.

A15Improvement Logic

Do not fix every small comment. Focus on repeated issues.

Write the issue clearly before deciding the fix.

Prioritise actions that improve the user experience most.

Assign an owner so the improvement is realistic.

Sample Improvement Plan

From feedback to action

Repeated issue: 2/3 testers did not notice when the crowd data was updated.
Fix: Add "last updated 5 minutes ago" label below each stall.
Priority: High because it affects trust and decision-making.
✍️ Sample answer guide

Positive feedback: Users liked seeing crowd level before leaving class.
Common issue: Users wanted more confidence that the information was current.
Fix: Add timestamp and refresh button label.
